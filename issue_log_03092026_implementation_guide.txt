P0-1 (Critical): Backtest trade accounting and cost attribution integrity
------------------------------------------------------------------------------
Problem:
- Entry trades are not recorded.
- Forced final exits are not recorded as trades.
- Forced final exits do not include slippage in totals.
- total_trades is undercounted and cost diagnostics can be biased.

Primary files:
- src/backtest/engine.py
- src/evaluation/backtest_metrics.py
- tests/test_backtest.py

Implementation steps:
1. Record entry trades explicitly in the engine.
   - In the branch where a new position is opened, append a Trade record.
   - Include timestamp, symbol, quantity, entry_price, entry_fee, entry_slip.

2. Record all exit trades explicitly.
   - Keep current close-on-signal-change trade append.
   - Add trade append for forced final close in the end-of-run block.

3. Ensure slippage is accumulated for every exit path.
   - In forced final close block, compute exit_slip and add to total_slippage.

4. Make total_trades derived from actual trade records.
   - Keep metadata["total_trades"] = len(all_trades), now accurate after 1-3.

5. Clarify cost semantics in metadata.
   - Add metadata fields:
     - entry_trades
     - exit_trades
     - round_trips_estimate
   - Use these in report formatting if desired.

6. Add/extend tests.
   - test_entry_trade_recorded
   - test_forced_final_exit_recorded
   - test_slippage_accumulated_on_forced_exit
   - test_total_trades_matches_trade_list

Acceptance criteria:
- For a deterministic synthetic signal path, trade count matches expected entries+exits.
- total_fees and total_slippage match hand-calculated values.
- Existing tests and new tests pass.

Verification commands:
- pytest -q tests/test_backtest.py
- python scripts/run_backtest.py --output-dir outputs/<version>

P0-2 (High): Feature selection pipeline non-compliant with v2 summary
------------------------------------------------------------------------------
Problem:
- Current screening is optional GRA-GRG top-k.
- Required workflow is:
  A) Pearson correlation filter (non-cross-market only)
  B) XGBoost importance ranking on validation
- Compact baseline and [screen]/[fixed] policy are not enforced as documented.

Primary files:
- src/features/builder.py
- src/features/screening.py
- scripts/train_xgboost.py
- configs/china_open_universe_minimal.template.toml
- tests/ (new feature screening tests)

Implementation steps:
1. Add feature-group tagging in builder.
   - Build explicit lists:
     - non_cross_market_candidates
     - cross_market_fixed
   - Keep cross-market/macro block untouched by screening logic.

2. Implement Pearson correlation filter utility.
   - New function in src/features/screening.py:
     - correlation_filter(df, cols, threshold)
   - Inputs only from training split to avoid leakage.
   - Keep one from each highly correlated pair using deterministic rule:
     - priority list based on interpretability and v2 keep list.

3. Implement validation-based XGBoost importance selector.
   - New function:
     - select_by_xgb_importance(train_df, val_df, cols, top_n or min_importance)
   - Use same model family as baseline training (light config for screening).
   - Return stable selected subset.

4. Integrate two-stage selection into training flow.
   - Do not pre-screen in run_pipeline.
   - In train_xgboost.py:
     - split data first
     - run Stage A and Stage B using train/val only
     - train final model on selected feature set
   - Save selected feature list artifact:
     - outputs/<version>/selected_features.json
     - include stage A dropped pairs and stage B ranking.

5. Add config controls.
   - Add to [features] or [model]:
     - feature_selection_enabled = true
     - corr_threshold = 0.9 (or 0.95)
     - non_cross_top_n = <int>
   - Keep backward compatibility defaults.

6. Add tests.
   - test_correlation_filter_drops_one_of_high_corr_pair
   - test_cross_market_features_not_filtered
   - test_selection_uses_train_val_only

Acceptance criteria:
- Selected non-cross feature set is deterministic.
- Cross-market/macro feature block remains unchanged.
- Artifacts contain full screening trace for reproducibility.

Verification commands:
- python scripts/run_pipeline.py --config configs/china_open_universe_minimal.template.toml
- python scripts/train_xgboost.py --config configs/china_open_universe_minimal.template.toml
- pytest -q

P0-3 (High): Threshold calibration objective mismatch with Phase 2/3
------------------------------------------------------------------------------
Problem:
- k is calibrated by a simple "edge" proxy.
- Required calibration target is post-cost Sharpe with turnover/trade-count sanity.

Primary files:
- src/evaluation/signal.py
- scripts/train_xgboost.py
- src/evaluation/backtest_metrics.py
- tests/ (new threshold calibration tests)

Implementation steps:
1. Introduce backtest-aware calibration function.
   - Add calibrate_k_post_cost(...) that for each k:
     - generates signals on validation set
     - runs backtest with configured fee/slippage
     - computes post-cost Sharpe
     - tracks turnover and trade count constraints

2. Add constraints in selection rule.
   - Example:
     - min_trade_count >= N
     - turnover <= max_turnover
   - If multiple k satisfy constraints, choose max Sharpe.
   - If none satisfy constraints, fallback to highest Sharpe and log warning.

3. Persist full calibration table.
   - Save outputs/<version>/threshold_calibration.csv with columns:
     - k, sharpe_post_cost, turnover, trades, cond_hit, total_return, selected

4. Update signal_config artifact.
   - Include calibration objective and constraints used.

5. Update training report.
   - Add calibration section with winning k rationale.

6. Add tests.
   - test_calibration_selects_best_post_cost_sharpe
   - test_constraints_respected
   - test_fallback_when_no_k_passes

Acceptance criteria:
- k selection is traceable to post-cost Sharpe table.
- signal_config and evaluation_report include calibration evidence.

Verification commands:
- python scripts/train_xgboost.py --config configs/china_open_universe_minimal.template.toml
- inspect outputs/<version>/threshold_calibration.csv
