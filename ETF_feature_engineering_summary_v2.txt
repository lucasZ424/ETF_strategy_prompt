Feature Engineering Summary (v2)
===============================

Design update:
- Forecast target moved to forward 5-day log return.
- Cross-market and macro context block is preserved as-is.
- Non-cross-market sections are compacted to reduce redundancy and improve
  generalization to unseen ETFs.
- Feature selection now uses a two-stage workflow:
  1. Pearson correlation filter on non-cross-market features.
  2. XGBoost importance ranking on validation results.

Feature selection implementation logic
--------------------------------------
A. Pearson correlation filter
- Apply only to non-cross-market features in the training set.
- Compute pairwise Pearson correlations.
- If |corr| >= threshold (recommended start: 0.90 or 0.95), remove one feature
  from the pair based on interpretability and later validation utility.

B. XGBoost importance ranking
- Train baseline XGBoost on the filtered feature set.
- Rank features using validation-based importance analysis.
- Keep features that show stable usefulness across validation windows and,
  ideally, across leave-one-ETF-out splits.

C. Practical note
- The compact baseline is the default starting point.
- [screen] features may be reintroduced only if they improve validation and
  out-of-sample robustness rather than only in-sample fit.
