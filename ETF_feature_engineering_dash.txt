ETF Feature Engineering - Dashboard Forecast Pipeline (Raw Close Target)
=======================================================================

Scope
-----
This file is exclusive to the dashboard forecast branch.
It is intentionally separate from the trade feature file.

Dashboard objective
-------------------
- One shared model artifact
- Multi-horizon outputs for 1d / 3d / 5d dashboard display
- Official target and displayed value = raw future Close price

Target definition (frozen)
--------------------------
For each symbol and date t:

    y_close_1d = Close_{t+1}
    y_close_3d = Close_{t+3}
    y_close_5d = Close_{t+5}

No log-return conversion is used in this dashboard branch.

Core modeling problem and redesign rule
---------------------------------------
If target is raw future close, using many direct price-level path features
(e.g., log_close, log_close_lag1/2/3/5/10) can make the model overfit to a
near-identity mapping and under-learn directional move information.

Therefore the feature design follows:
1. Keep exactly one explicit level anchor (`close_anchor`) so the pooled model
   can map predictions to the correct nominal price scale per ETF.
2. Shift most remaining features to normalized/stationary dynamics
   (returns, ratios, volatility, candle shape, volume state).
3. Keep strict no-lookahead construction (only data known at date t).

Notation
--------
- C_t = raw close at date t
- O_t = open, H_t = high, L_t = low, V_t = volume
- eps = 1e-8 numerical safety constant

Feature blocks
--------------

1) Anchor + return dynamics
---------------------------
- close_anchor
  Formula: C_t
  Role: only explicit level anchor for raw-close target scale

- ret1_close
  Formula: log(C_t / C_{t-1})

- ret3_close
  Formula: log(C_t / C_{t-3})

- ret5_close
  Formula: log(C_t / C_{t-5})

- ret10_close
  Formula: log(C_t / C_{t-10})

- ret1_lag1, ret1_lag2, ret1_lag3, ret1_lag5
  Formula: lagged values of ret1_close

- ret3_lag1
  Formula: lagged value of ret3_close

- ret5_lag1
  Formula: lagged value of ret5_close

- ret_accel_1v5
  Formula: ret1_close - ret5_close / 5
  Role: short-vs-medium speed change

- ret_accel_3v10
  Formula: ret3_close / 3 - ret10_close / 10
  Role: medium-vs-broader speed change

2) Trend-shape
--------------
- close_over_ma5
  Formula: C_t / MA5(C)

- close_over_ma10
  Formula: C_t / MA10(C)

- close_over_ma20
  Formula: C_t / MA20(C)

- ema5_over_ema20
  Formula: EMA5(C) / EMA20(C)

- ma5_over_ma20
  Formula: MA5(C) / MA20(C)

- slope5_close
  Formula: rolling OLS slope of log(C) over last 5 bars

- slope10_close
  Formula: rolling OLS slope of log(C) over last 10 bars

3) Volatility + candle shape
----------------------------
- rv5_close
  Formula: rolling std(ret1_close, 5)

- rv10_close
  Formula: rolling std(ret1_close, 10)

- rv5_over_rv20
  Formula: rv5_close / (rolling std(ret1_close, 20) + eps)

- atr14_over_close
  Formula: ATR(14) / C_t

- hl_range_close
  Formula: (H_t - L_t) / (C_t + eps)

- gap_oc_prev_close
  Formula: log(O_t / C_{t-1})

- oc_body_pct
  Formula: (C_t - O_t) / (C_t + eps)

- upper_shadow_pct
  Formula: (H_t - max(O_t, C_t)) / (C_t + eps)

- lower_shadow_pct
  Formula: (min(O_t, C_t) - L_t) / (C_t + eps)

4) Volume state
---------------
- vol_z20
  Formula: (V_t - MA20(V)) / (STD20(V) + eps)

- vol_ma_ratio20
  Formula: V_t / (MA20(V) + eps)

- vol_chg1
  Formula: log((V_t + 1) / (V_{t-1} + 1))

5) Cross-market context (already aligned, lag-safe)
----------------------------------------------------
From cross-market alignment module, merged by date:
- spy_ret_lag1
- qqq_ret_lag1
- ieur_ret_lag1
- spy_ret10d_lag1
- qqq_ret10d_lag1
- ieur_ret10d_lag1
- vix_chg_lag1
- us10y_chg_lag1
- dxy_ret_lag1

6) Calendar (cyclical encoding)
--------------------------------
- dow_sin, dow_cos
  Formula: sin/cos(2*pi*dow/5), dow in [0..4]

- month_sin, month_cos
  Formula: sin/cos(2*pi*(month-1)/12), month in [1..12]

Selection and validation rules
------------------------------
A. Pearson correlation filter within dashboard features only
- Start threshold: |corr| >= 0.90

B. Feature importance ranking (XGBoost gain)
- Apply after Pearson filter
- Keep top-k by config if needed

C. Horizon robustness check
- Prefer features that are stable across 1d/3d/5d tasks

D. Leakage guard
- Use only information available at t for predicting t+h
- Cross-market features must remain strict lag-aligned to China session

Frozen decisions reflected here
-------------------------------
1. Dashboard branch is separate from trade feature design.
2. Dashboard target is raw future close (1d/3d/5d).
3. Shared pooled model remains preferred unless performance proves inadequate.
4. Feature design uses one explicit price anchor + mostly normalized dynamics.
